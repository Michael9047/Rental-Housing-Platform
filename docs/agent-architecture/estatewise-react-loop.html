一次完整的 ReAct 追踪
Step 0 · 任务注入 & 上下文组装
Supervisor DAG 把子任务 + 可用工具列表 + 系统提示词一起注入 Agent
const context = { task: "搜索 Chapel Hill 3b2b 房源，预算 $2000", tools: ['property_search', 'neo4j_query', 'pinecone_similar', 'commute_calc'], systemPrompt: "你是房产搜索专家...", maxSteps: 10, tokenBudget: 8192, timeoutMs: 30000 }; agent.run(context);
Step 1 · 思考 Thought
LLM 分析当前上下文，决定是调用工具还是直接回答。输出 JSON 结构化思考。
// LLM 输出（结构化 JSON） { "thought": "需要先用 property_search 工具查询 Chapel Hill 地区的 3b2b 房源", "action": "call_tool", // call_tool | answer | ask_clarify "tool": "mcp:property_search", "args": { city: "Chapel Hill", beds: 3, baths: 2, budget: 2000 }, "confidence": 0.87 }
Step 2 · 行动 Action
Agent 通过 MCP 协议调用工具，执行真实的 I/O（数据库查询、API 调用、知识图谱遍历）
// MCP 工具调用 const result = await mcpClient.callTool('property_search', { city: 'Chapel Hill', beds: 3, baths: 2, budget: 2000 }); // → 返回 12 条匹配房源 // → 耗时 1.4s，消耗 487 tokens
Step 3 · 观察 Observation
工具结果被追加到对话上下文，作为下一轮 LLM 思考的输入。Agent 判断是否继续。
// 观察结果追加到上下文 context.messages.push({ role: 'tool', content: `找到 12 条房源： 1. 123 Main St — $1,800/月 — 3b2b — 评分 4.2 2. 456 Oak Ave — $1,950/月 — 3b2b — 评分 4.7 ... 12. 789 Pine Rd — $2,100/月 — 3b2b — 评分 3.9` }); // Agent 判断：还需进一步筛选 → 继续循环 if (!done && step < maxSteps && tokens < budget) goto Step 1;
Step 4 · 思考 Thought (第二轮)
LLM 根据观察结果调整策略。发现结果太多，决定用 Pinecone 向量搜索做语义重排，同时查通勤时间。
{ "thought": "12 条结果太多。用 Pinecone 做语义相似度重排，同时查通勤到 UNC 的时间", "action": "call_tool", "tool": "mcp:pinecone_similar", // 本轮先用 Pinecone "args": { query: "安静的 3b2b 近 UNC 校园", topK: 6 } }
Step 5–6 · 并行行动
EstateWise 支持单轮并行调用多个 MCP 工具——同一轮 thought 可以触发两个 action，结果合并后统一观察
// 并行调用（同一轮） const [pineconeRes, commuteRes] = await Promise.all([ mcpClient.callTool('pinecone_similar', { query, topK: 6 }), mcpClient.callTool('commute_calc', { origin: addr, dest: 'UNC Chapel Hill' }) ]); // pineconeRes → 语义 Top 6，已按相似度降序 // commuteRes → 每套房到 UNC 的通勤时间（分钟）
Step 7 · 思考 Thought (第三轮) — 决策终止
综合所有信息后，LLM 判断已充分回答，输出 action=answer，终止循环。
{ "thought": "已有语义排名 + 通勤数据，可以给出 Top 3 推荐并附理由", "action": "answer", // ← 终止信号 "confidence": 0.94 } // 循环退出 → 生成最终回复
Step 8 · 最终回复 · Final Answer
LLM 把所有观察结果 + 上下文综合成最终自然语言回复，返回给 Supervisor
return await llm.finalAnswer(context); // → "为您推荐 Chapel Hill 3 套最佳 3b2b 房源： // 1. 456 Oak Ave — $1,950/月 — 通勤 12min — ★4.7 （最推荐） // 2. 123 Main St — $1,800/月 — 通勤 18min — ★4.2 // 3. 234 Elm St — $2,100/月 — 通勤 8min — ★4.5 // 需要预约看房吗？"
我们的 Agent vs EstateWise ReAct

🏠 我们当前：线性 Pipeline

  • 单次 LLM 调用 → 输出 JSON
  • 工具调用在 LLM 外部（代码硬编排）
  • 无循环——一次失败无法重试
  • 无 token 预算 / 超时守卫
  • 无观察反馈——LLM 看不见工具结果
  • 优势：快、简洁、可预测

🧠 EstateWise：ReAct 循环

  • LLM ← → 工具 多轮交互
  • LLM 自主决定调哪个工具、何时停
  • 观察驱动：失败可重试、结果不足可追加
  • 三层守卫：token / timeout / maxSteps
  • 支持单轮并行调用多个 MCP 工具
  • 代价：tokens 消耗多 3-5×，延迟更高

ReAct 循环 = 思考(Thought) → 行动(Action) → 观察(Observation) → 循环，直到 LLM 输出 action=answer 或触发守卫上限
我们的项目可在下一阶段 Agent 迭代时，为基础查询保留线性 Pipeline，复杂多步请求引入 ReAct Loop
