# 团队协作速查

> 租房平台 · 6人团队 · Vibe Coding 模式

---

## 一图流

```
你 vibe 改代码 → AI 建 Issue + 发 PR → review → 合并 → 自动完成
```

---

## 日常操作（3 条命令搞定）

```bash
# 1. 开始干活
git checkout main && git pull && git checkout -b feat/你的功能名

# 2. 改完代码后，告诉 AI：
"提交代码，按规范建 Issue 并发 PR"

# 3. PR 通过、合并后
git checkout main && git pull && git branch -d feat/你的功能名
```

---

## PR 要求

| 项目 | 要求 |
|------|------|
| 标题 | `[模块] 简述`（中文） |
| Issue | 必须关联，描述中写 `Closes #X` |
| 前端截图 | **必填**，AI 发 PR 时标"待补充"，review 前补上 |
| AI 声明 | PR 模板中勾选用了什么 AI 工具 |
| CI | 必须全绿 |

---

## 自动化的部分（你不用管）

- 新 Issue/PR → 自动加入看板
- PR 分配 reviewer → CODEOWNERS 自动匹配
- PR 合并 → Issue 自动关闭，看板卡片自动移 Done

---

## Commit 格式

```
feat(map): 接入高德地图地理编码
fix(payment): 修复微信支付回调签名失败
```

## 团队分工（5 业务板块 + 1 美工）

| 角色 | 成员 | 负责内容 |
|------|------|----------|
| 房源核心 | 成员1 | 房源 CRUD、图片管理、数据导入、首页、搜索页（前后端） |
| 交易流程 | 成员2 | 预约看房、支付、电子合同（前后端） |
| 地图与搜索 | 成员3 | 高德地图、周边 POI、语义搜索、AI 搜索（前后端） |
| 用户与通讯 | 成员4 | 登录注册、即时通讯、邮件/短信通知（前后端） |
| 管理后台 | 成员5 | 管理员面板、CI/CD、Docker、规范文档（前后端） |
| 🎨 美工/设计 | 成员6 | 全局样式、设计规范、所有前端页面的视觉 review |

> **美工的工作方式**：只负责自己的设计产出（全局样式、设计规范、布局组件），PR 由 team-lead review。美工不审查其他人的代码。
> **微信相关**：暂不开发，相关文件归共享池维护。
