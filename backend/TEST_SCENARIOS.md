## 七、多轮对话场景（验证漏斗状态传递）

### 场景 A：从模糊到精确的完整找房流程

```
用户: "我想在苏州租房"
Agent: [explore] 市场概览，引导细化 → 追问区域/预算/户型

用户: "工业园区吧，预算2000左右"
Agent: [calibrate] 园区2000元能租到什么样的 → 展示价格分布

用户: "我想要一居室，最好离地铁近"
Agent: [narrow] 地铁沿线一居室筛选 → 列出候选

用户: "帮我对比一下第一套和第三套"
Agent: [compare] 对比分析 → 给出推荐

用户: "就第一套吧，怎么预约看房"
Agent: [decide] 引导预约流程
```

### 场景 B：学生租房（学校周边 + 预算敏感）

```
用户: "西交利物浦大学附近有房子吗"
→ search, extract: district=苏州, near=西交利物浦大学

用户: "太贵了，有没有1000以内的合租"
→ search, query_rewrite: price_max=1000, type=shared

用户: "离学校走路能到的"
→ search + commute: mode=walking, max_time=20min

用户: "押金要多少？"
→ faq: 押金政策
```
