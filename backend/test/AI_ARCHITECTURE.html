2. 真实执行链路

Dispatcher 扁平分派（唯一活跃路径）

agent.py  POST /api/v1/agent/sessions/:id/messages
  │
  ▼
dispatcher.py  dispatch(session_id, message, db)
  │
  ├─ router.classify_message()  ← LLM 单次调用 → intent + stage + complexity
  │
  ├─ intent == "search"
  │    └─ SearchAgent.search()  → extract → 检索 → 通勤 → 评分 → LLM 推荐
  │
  ├─ intent == "compare"
  │    └─ CompareAgent.compare()  → 7维评分 → LLM 解释
  │
  ├─ intent == "manage_cart"
  │    └─ CartService (纯 CRUD, 非 Agent)
  │
  ├─ intent == "faq"
  │    └─ agent_faq.match_faq() (关键词匹配, 非 Agent)
  │
  └─ intent == "general"
       └─ llm_service.complete_text() (通用 LLM 兜底)
⚠️ 不存在 DAG 拓扑排序、不存在 Handoff 交接链、不存在 ReAct Tool Loop。
Dispatcher 的文档开头就写了："替代 Supervisor + DAG + AgentRegistry 的复杂编排链路"——旧架构被替换但代码未删除。
