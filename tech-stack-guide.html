🏠 租房匹配系统
Phase 1-7 技术栈全解

Phase 1 · Foundation 100% 完成

🐍 FastAPI

是什么：Python 的现代 Web 框架，用来写后端 API 接口。

为什么用它：代码量少、性能高（异步）、自动生成 API 文档，是目前 Python 后端最火的选择。就像 Flask 的升级版，但自带数据校验、自动文档、类型提示。

主流程度：GitHub 70k+ stars，Netflix/Uber/Microsoft 都在用。

🗄️ PostgreSQL 16

是什么：开源的关系型数据库，存所有数据（用户、房源、预约等）。

为什么用它：它是目前功能最强大的开源数据库——支持 JSON、全文搜索、地理坐标，还能装扩展（比如 pgvector 做 AI 向量搜索）。MySQL 的升级替代品。

类比：如果 MySQL 是 Excel，PostgreSQL 就是 Excel + Python + 地图。

🔌 pgvector

是什么：PostgreSQL 的一个扩展插件，让数据库能存「向量」并做相似度搜索。

向量是什么：把一段文字（比如房源描述）用 AI 模型转成一串数字（比如 1536 个小数），这段数字就是向量。意思相近的文字，向量也相近。

为什么不用 Elasticsearch：pgvector 直接嵌在 PostgreSQL 里，不用另外装一套搜索引擎，运维简单，一个 SQL 语句就能同时做向量搜索+普通筛选。

🔄 SQLAlchemy 2.0 + Alembic

SQLAlchemy 是什么：Python 的 ORM（对象关系映射）。让你用 Python 代码操作数据库，不用手写 SQL。

Alembic 是什么：数据库迁移工具。每次改表结构（比如加个字段），自动生成升级/回滚脚本，像 Git 管理代码一样管理数据库版本。

主流程度：Python 生态绝对标准，几乎没有替代品。

🔐 JWT + bcrypt

JWT：JSON Web Token。用户登录后，服务端发一个加密的"令牌"给前端，之后每次请求带这个令牌来证明身份。无状态、不需要服务端存 session。

bcrypt：密码哈希算法。用户密码不会明文存数据库，而是 bcrypt 加密后再存。即使数据库泄露，密码也无法还原。

🐳 Docker Compose

是什么：用 YAML 文件一键启动所有服务（PostgreSQL + Redis）。不用手动安装数据库，`docker compose up` 就够了。

Phase 2 · Core Features 100% 完成

🖼️ Vue 3 + Composition API

是什么：前端框架。写网页界面的。Vue 3 是最新大版本，Composition API 是它的新写法（比旧版 Options API 更灵活）。

为什么选 Vue 而不是 React：Vue 的模板语法（v-if、v-for）更像 HTML，学习曲线低。中文文档质量高，社区活跃。对新手更友好。

对比：React 用 JSX（HTML 写在 JS 里），Vue 用模板（HTML + 指令），更像传统 Web 开发。

🧩 Element Plus

是什么：Vue 3 的 UI 组件库。提供按钮、表格、弹窗、表单等现成组件。

为什么：不用自己写 CSS。`` 就是按钮，`` 就是表格。饿了么团队出品，中文文档一流。

⚡ Vite

是什么：前端构建工具。开发时热更新极快（改代码秒级刷新），打包时输出优化过的文件。

为什么不用 Webpack：Vite 更快。Webpack 是上一代标准，Vite 是新一代标准。Vue 官方默认推荐。

📦 Pinia

是什么：Vue 的状态管理库。管理全局数据（比如当前登录用户信息、搜索结果），让不同页面共享数据。

为什么不用 Vuex：Pinia 是 Vuex 的继任者，API 更简洁、TypeScript 支持更好。Vue 官方推荐。

📘 TypeScript

是什么：JavaScript 的超集，加了类型系统。写代码时能自动提示、检查错误，减少 bug。

通俗理解：JavaScript 是铅笔手绘，TypeScript 是 AutoCAD 制图——多了一步标注，但更严谨、更适合大型项目。

Phase 3 · Embedding Pipeline 80% 完成

🧠 text-embedding-3-small（OpenAI）

是什么：OpenAI 提供的一个 AI 模型，专门把文字转成向量数字。

怎么工作：输入「两室一厅 近地铁」，输出 1536 个小数组成的向量。跟它意思相近的房源向量也会靠近。

为什么选它：便宜（$0.02/百万字）、效果好、1536 维是行业标准维度。

类比：这个模型就像一个「翻译官」，把中文房源描述翻译成数学语言（向量），让计算机能理解语义相似度。

📨 Celery + Redis

Celery 是什么：Python 的异步任务队列。把耗时操作（比如调 OpenAI 生成向量）放到后台执行，不阻塞用户请求。

Redis 是什么：内存数据库，极快。在这里充当 Celery 的「消息中介」——Celery 通过 Redis 收发任务。

为什么：调 OpenAI API 可能需要几百毫秒，如果同步等待，用户会感觉卡。Celery 把它变成后台任务，API 立即返回。

📊 EmbeddingJob

是什么：一个数据表，记录每次生成向量的状态（pending → processing → completed / failed）。

为什么：方便管理员查看哪些房源还没生成向量、哪些失败了需要重试。

Phase 4 · Semantic Search 100% 完成

🔍 IVFFlat 索引

是什么：pgvector 提供的向量索引算法。给向量数据建立索引，加速相似度搜索。

原理（通俗）：相当于给 10000 个向量先分组聚类成 100 个中心，搜索时只比较最近的几个组，不用全扫一遍。

为什么不用 HNSW：IVFFlat 构建快、内存省。HNSW 搜索更快但占用更多内存，等数据量到 10 万+再切换。

📐 L2 Distance（欧氏距离）

是什么：衡量两个向量相似度的数学公式。距离越小，越相似。

通俗理解：把两个向量的 1536 个维度想象成空间中的两个点，L2 就是这两点之间的直线距离。

🔀 混合搜索（Hybrid Search）

是什么：一次请求同时做两件事：向量相似度匹配 + 传统条件筛选（区域/价格/户型），在一个 SQL 查询里完成。

示例：用户搜「园区 两室 便宜」→ 向量匹配「园区两室便宜」语义 → 同时过滤 district=园区, bedrooms=2 → 按相似度排序返回。

Phase 6 · Booking & Notifications 100% 完成

🔁 状态机模式（Booking Status）

是什么：预约的 5 种状态有严格的流转规则：pending（待审批）→ approved（通过）/ rejected（拒绝）/ cancelled（取消）→ completed（完成）。

为什么：防止非法状态跳转（比如已取消的预约不能被审批）。用 Python Enum + 数据库 Enum 双重约束。

🔔 站内通知（Pull 模式）

是什么：通知存在数据库里，前端定时或用户点击时拉取最新通知。

为什么不用 Push（WebSocket）：这是一个 B 端租房管理系统，实时性要求不高。Pull 模式更简单、更稳定，不需要维护长连接。

🔴 el-badge（未读数红点）

是什么：Element Plus 的徽标组件。导航栏铃铛图标上显示未读通知数量的小红点。

Phase 7 · Admin Panel 100% 完成

📈 Dashboard 统计（StatsService）

是什么：后端聚合查询，统计系统全局数据——用户总数、房源总数、预约数、各区域房源分布。

实现：用 SQLAlchemy 的 `func.count()` 和 `group_by` 做聚合查询，一次请求返回所有统计。

📝 审计日志（AuditLog）

是什么：记录每个管理操作的日志表——谁在什么时候做了什么（改用户角色、审核房源、触发重建索引）。

为什么需要：可追溯、可复盘。出了问题能查到是谁操作的。PostgreSQL JSON 列灵活存储操作详情。

🛡️ 路由守卫（requiresAdmin）

是什么：Vue Router 的 beforeEach 钩子。访问 `/admin/*` 页面时，检查用户角色是不是 admin，不是就踢回首页。
