A single FastAPI application (`app/main.py::create_app`) is the only process; it mounts shared middleware (CORS, Prometheus metrics, Redis-backed rate limiting, request logging, global exception handlers) and includes every feature's `APIRouter` under `/api/v1` via `app/api/v1/router.py`, which is the sole cross-child wiring point. All children share one SQLAlchemy async engine plus Alembic migrations (`alembic/versions/*`) defined in `app/db/base.py`, and they communicate with each other indirectly through the database rather than direct imports — for example, bookings and contracts reference user/property models from their own modules but never import each other's services. Long-running work (embedding generation, notification dispatch, payment callbacks, CSV import) is offloaded to Celery workers configured by `app/celery_app.py`. Cross-cutting concerns are centralized in `app/core/`: `config.Settings` (pydantic-settings loaded from `.env`), `security` (JWT + WeChat token helpers), `logging`, `monitoring`, and `security_audit` (rate limiter). Third-party integrations (OpenAI/DeepSeek LLMs, pgvector embeddings, Amap geocoding, WeChat Mini Program, Alipay/WeChat Pay, Alibaba SMS, SMTP email) are all consumed through dedicated service classes in `app/services/` so routes stay thin.
