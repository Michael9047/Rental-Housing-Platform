Single-page Vue 3 `<script setup>` component (`AiSearch.vue`) drives a three-phase state machine (`input` → `form` → `results`) controlled by the local `phase` ref. Parsing is delegated to `aiSearchService.parse`, which POSTs to `/ai-search/parse`; if the returned `CompletenessReport.is_complete` is false the view switches to the form phase and renders `MissingField` entries as Element Plus inputs (district select, price range, bedrooms, property_type). When complete — or after manual form submission via `handleSearch` — the view calls `aiSearchService.search` (`POST /ai-search/search`) and displays the AI summary plus a horizontal scroll of `PropertyCard` mini-cards. The shared chat data model lives in `types/chat.ts` (`ChatSession`, `ChatMessage`, `MatchedProperty`, `SSEEvent`) and is reused by other chat features; this module only imports it for type compatibility. All HTTP calls go through the shared `@/services/api` client, so this module depends on no backend-specific code beyond the two REST endpoints.
