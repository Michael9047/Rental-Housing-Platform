Four sibling Vue 3 `<script setup lang="ts">` components under `src/views/`, each a self-contained page with inline scoped CSS. Cross-cutting state lives in Pinia stores (`usePropertyStore` from `@/stores/property`, `useAuthStore` from `@/stores/auth`) consumed via `storeToRefs`; bookings use the dedicated `bookingService.list()` / `updateStatus()` API client instead of a store. Navigation is driven by `vue-router` — `ManageProperties.vue` pushes to `/property/create`, `/property/:id/edit`, and `/property/:id/images`, while `CreateProperty.vue` toggles create vs edit mode by checking `route.name === 'edit-property'`. Data fetching happens in `onMounted`; error feedback goes through `ElMessage`/`ElMessageBox` and global HTTP interceptors swallow caught errors. The image view reads `route.params.id` directly rather than relying on a shared route prop.
