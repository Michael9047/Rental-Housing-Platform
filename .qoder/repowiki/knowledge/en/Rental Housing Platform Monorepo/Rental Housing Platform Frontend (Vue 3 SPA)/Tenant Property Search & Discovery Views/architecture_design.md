Four sibling Vue 3 `<script setup lang="ts">` components under `src/views/`, each one a top-level route page with its own scoped CSS:
- `Home.vue` — landing page: hero voice/AI search bar, region quick-links, and a featured-property carousel; navigates to `/search` or `/property/:id`.
- `Search.vue` — full-text + faceted search page. A reactive `filters` object (country/district/overseas_area, price range, bedrooms, property_type) is mapped into `PropertySearchParams` and dispatched through `usePropertyStore.fetchSearch`; results are client-side paged by `pageSize=6`. Route query (`q`, `country`, `district`) is mirrored bidirectionally via `watch(() => route.query)`.
- `PropertyDetail.vue` — detail page for a single listing. Reads `currentProperty` from Pinia via `storeToRefs`, fetches POI analysis and similar listings directly through `@/services/property`, embeds an OpenStreetMap iframe, and renders mock reviews.
- `MapSearch.vue` — standalone Leaflet map view. Owns the `L.Map` instance, loads viewport-bounded properties via `@/services/map.getPropertiesInBounds`, implements manual grid-based clustering below zoom 13, and geocodes city names against Nominatim.

Cross-cutting concerns are delegated outward: UI primitives come from Element Plus (`el-input`, `el-card`, `el-pagination`, `el-carousel`, `el-rate`, `ElMessage`), shared state lives in Pinia stores (`usePropertyStore`, `useAuthStore`), navigation uses `vue-router`, and data fetching goes through typed service modules (`@/services/property`, `@/services/map`). The only cross-view coupling is the shared `BookingDateDialog` component used to initiate bookings from any listing card.
