Standard WeChat miniprogram layout rooted at `app.json`/`app.js`, with four tabBar pages (`index`, `chat`, `booking`, `me`) plus two non-tab pages (`search`, `property`). Shared UI is factored into reusable components under `components/` (`map-view`, `property-card`, `search-bar`), each following the native `.js/.json/.wxml/.wxss` quad. Cross-cutting concerns live in `utils/`: `api.js` wraps `wx.request` as a Promise-based client that prepends `app.globalData.baseUrl`, injects a Bearer token from `access_token`, and centralizes 401 logout + toast handling; `auth.js` orchestrates `wx.login` → backend `/wechat/auth/wechat/login` → storage of `access_token` and `user`. Pages consume these via `require()` and call REST endpoints against the FastAPI backend at `http://localhost:8000/api/v1`. The `pages/booking/detail/` sub-page demonstrates nested routing within a tab page. Permissions for location are declared in `app.json` to support the map view.
